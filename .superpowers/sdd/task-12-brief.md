# Task 12: Prep Calendar UI

## Overview

Visual calendar for viewing and managing interview prep tasks. Displays events generated by Task 11 calendar generation system.

## Requirements

### Calendar Views
- **Monthly view** (traditional calendar grid)
- **Weekly view** (day-by-day breakdown)
- **List view** (chronological task list)
- **Toggle between views** with buttons/tabs

### Filtering
- **3 filter modes**:
  - Interview-specific (show only tasks tied to upcoming interviews)
  - General (show only evergreen prep tasks)
  - All (show everything)
- Filters apply across all views

### Task Interactions
- **Mark complete**: Click task → toggle isCompleted
- **Snooze task**: Click → set snoozeUntil date (1 day, 1 week, 2 weeks, or custom)
- **Add custom event**: Modal form to create new prep task
- **Delete task**: Confirm & remove from calendar

### Prep Intensity Indicator
- Visual indicator showing prep intensity for the week/month
- Color scale: green (light 0-5h), yellow (moderate 5-15h), orange (heavy 15-25h), red (intense >25h)
- Shows total hours per week

### Color Coding
- **Blue**: Interview-specific prep tasks
- **Green**: General evergreen prep tasks
- **Gray**: Custom user-added tasks

### States
- **Loading state**: While fetching calendar events
- **Error state**: With retry button
- **Empty state**: "No prep tasks scheduled"

## Files to Create

- `app/(dashboard)/prep-calendar/page.tsx` - Main page component
- `components/CalendarView.tsx` - Calendar display (monthly/weekly/list)
- `components/CalendarEventForm.tsx` - Modal form for creating/editing events

## API Endpoints to Use

All endpoints from Task 11:
- `GET /api/calendar?type=interview|general|all&fromDate=&toDate=` - Fetch events
- `POST /api/calendar` - Create custom event
- `PUT /api/calendar/[id]` - Update event (mark complete, snooze)
- `DELETE /api/calendar/[id]` - Delete event

## Component Structure

### CalendarView.tsx
- Props:
  - `events: PrepCalendarEvent[]`
  - `view: "month" | "week" | "list"`
  - `filterMode: "interview" | "general" | "all"`
  - `onEventClick: (event) => void`

- Renders:
  - Monthly: Grid of weeks/days with events
  - Weekly: 7-day breakdown with detailed hourly breakdown
  - List: Sorted by startDate, expandable for details

### CalendarEventForm.tsx
- Modal with form fields:
  - Title (required)
  - Description (optional)
  - Start date (required)
  - End date (required, >= start date)
  - Category (dropdown: system-design, leetcode, behavioral, portfolio, etc.)
  - Hours estimate (number)
- Actions: Save, Cancel
- On save: POST /api/calendar, add to state
- On cancel: Close modal

## Technical Patterns

- Use React hooks for view/filter state management
- Fetch calendar events on mount and after mutations
- Use `date-fns` for date formatting and math
- Calendar grid layout: CSS Grid or Flexbox
- Color coding via className mapping (TailwindCSS)
- Modal: Simple div with overlay, close on escape/outside click
- All times in UTC from API, format for display using user's locale

## UI Details

### Monthly View
- Grid: 7 columns (Sun-Sat), rows for weeks
- Each day cell shows date + list of events (truncate if >2)
- Click day to see details or add event
- Current day highlighted
- Event dots/badges colored by type

### Weekly View
- 7 columns (days), each with hourly breakdown
- Show 24-hour timeline with events positioned by time
- Display total hours per day
- Prep intensity bar at top

### List View
- Table format: Date | Task | Category | Hours | Status
- Sortable by date, category, status
- Expandable rows for description
- Search/filter by title

### Prep Intensity Indicator
- Weekly summary card: "Week of Jul 21-27"
- Visual bar graph: total hours
- Color based on intensity level
- Show breakdown by type (interview vs general)

## Notes

- Snoozing tasks should remove them from current view until snoozeUntil date
- Completed tasks can be hidden or shown with a filter toggle
- Custom events can be any category (not restricted to templates)
- Calendar should be responsive: mobile = list view, desktop = monthly view default
- Integrates with Task 11 generation system (no hardcoded data)
- Authentication: already handled by layout middleware
